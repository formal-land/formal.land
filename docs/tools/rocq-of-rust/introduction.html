How it worksâ€‹

It converts Rust programs to the ğŸ“Â Rocq proof system. You access the source on GitHub. rocq-of-rust generates a shallow embedding of Rust into Rocq. We run the translation from the THIR level of the Rust compiler. The generated Rocq code is more verbose than the source Rust as we explicit all the low-level details, such as the sequencing of effects or the implicit borrowing/dereferencing.

Getting startedâ€‹

To install rocq-of-rust, you need to clone the GitHub repository and to have a working installation of Rust. Then, from the root folder of the project, you can run:

cargo install --path lib/

It will install the additional cargo rocq-of-rust command. Then to translate in Rocq the project of your choice:

cd my_rust_project/
# You need to copy the rust-toolchain file to force the right version of Rust
cp ../rocq-of-rust/rust-toolchain ./
cargo rocq-of-rust

This should generate a bunch of .v Rocq files, with one file per corresponding Rust file. To compile them, the simplest is to copy them in the RocqOfRust/ folder of the rocq-of-rust project and to run from there:

# Install the Rocq dependencies
opam repo add rocq-released https://rocq-prover.org/opam/released
opam install . --deps-only
# Compile the whole project (it takes time)
make -j4 # 4 is your number of cores

For robustness, the files are not dependent for compilation, so that one translation failure does not block you.

Workflowâ€‹

To formally verify a Rust project using rocq-of-rust we work as follows:

  1. translate the Rust code to Rocq using rocq-of-rust
  2. define the memory of the program (how the values will be allocated)
  3. write simulation functions for each function of the Rust code, to give a simpler and more functional definition of the code
  4. verify that these simulations are equivalent to the source code
  5. prove properties over these simulations

As some of the work is very verbose and repetitive, such as the definition of the simulation functions, but thankfully generative AI tools such as Github Copilot are better and better at code generation.
