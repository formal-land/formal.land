Logical Connectives

Beyond implication, logic has other connectives:

  • A /\ B — Conjunction (A and B)
  • A \/ B — Disjunction (A or B)
  • ~A — Negation (not A)
  • A <-> B — Biconditional (A iff B)

Conjunction: A /\ B

To prove A /\ B, prove both. To use it, extract both parts.

conjunction.v
(* Proving a conjunction *)
Theorem and_intro : forall A B : Prop, A -> B -> A /\ B.
Proof.
  intros A B HA HB.
  split.          (* Two subgoals: A and B *)
  - exact HA.
  - exact HB.
Qed.

(* Using a conjunction *)
Theorem and_elim_l : forall A B : Prop, A /\ B -> A.
Proof.
  intros A B [HA HB].   (* Destruct in intro! *)
  exact HA.
Qed.

(* Conjunction is commutative *)
Theorem and_comm : forall A B : Prop, A /\ B -> B /\ A.
Proof.
  intros A B [HA HB].
  split; assumption.    (* Prove both with assumption *)
Qed.

Tactics for /\

  • split — prove both parts
  • destruct H as [H1 H2] — break apart

Understanding Destruct Patterns

The pattern you use with destruct depends on the structure of what you're destructing:

For Conjunction (/\): Use [H1 H2]

A conjunction A /\ B contains both A and B. The pattern [H1 H2] extracts both parts simultaneously:

destruct_and.v
(* H : A /\ B *)
destruct H as [HA HB].
(* Now we have: HA : A  and  HB : B *)

(* Or directly in intro: *)
intros [HA HB].   (* Same effect *)

For Disjunction (\/): Use [HA | HB]

A disjunction A \/ B contains either A or B (not both). The pattern [HA | HB] creates two separate proof branches — one where you have A, and one where you have B:

destruct_or.v
(* H : A \/ B *)
destruct H as [HA | HB].
(* Creates TWO subgoals:
   - First subgoal: HA : A
   - Second subgoal: HB : B  *)

Memory Trick

Space between names = both at once (conjunction)
Bar | between names = separate cases (disjunction)

Think: [H1 H2] is "H1 AND H2", [HA | HB] is "HA OR HB"

Disjunction: A \/ B

To prove A \/ B, prove one side. To use it, consider both cases.

disjunction.v
(* Proving disjunction - choose a side *)
Theorem or_intro_l : forall A B : Prop, A -> A \/ B.
Proof.
  intros A B HA.
  left.           (* Choose left side *)
  exact HA.
Qed.

Theorem or_intro_r : forall A B : Prop, B -> A \/ B.
Proof.
  intros A B HB.
  right.          (* Choose right side *)
  exact HB.
Qed.

(* Using disjunction - case split *)
Theorem or_comm : forall A B : Prop, A \/ B -> B \/ A.
Proof.
  intros A B [HA | HB].  (* Case split *)
  - right. exact HA.
  - left. exact HB.
Qed.

(* Or elimination *)
Theorem or_elim : forall A B C : Prop,
  (A -> C) -> (B -> C) -> A \/ B -> C.
Proof.
  intros A B C HAC HBC [HA | HB].
  - apply HAC. exact HA.
  - apply HBC. exact HB.
Qed.

Negation: ~A

Negation ~A is defined as A -> False. If A leads to contradiction, A is false.

negation.v
(* ~A = A -> False *)
Print not.   (* fun A : Prop => A -> False *)

(* Ex falso: from False, anything follows *)
Theorem ex_falso : forall P : Prop, False -> P.
Proof.
  intros P HFalse.
  destruct HFalse.   (* False has no constructors! *)
Qed.

(* Contradiction: A and ~A give False *)
Theorem contradiction : forall A : Prop, A -> ~A -> False.
Proof.
  intros A HA HNA.
  apply HNA.     (* ~A = A -> False *)
  exact HA.
Qed.

(* Double negation introduction *)
Theorem double_neg_intro : forall A : Prop, A -> ~~A.
Proof.
  intros A HA HNA.   (* ~~A = ~A -> False *)
  apply HNA. exact HA.
Qed.

Important!

In intuitionistic logic (Rocq's default), we CANNOT prove ~~A -> A! This requires classical logic.

The exfalso Tactic

The exfalso tactic applies the principle "from False, anything follows" (ex falso quodlibet). It changes your goal to False.

exfalso.v
(* When you have a contradiction, use exfalso *)
Theorem using_exfalso : forall A B : Prop, A -> ~A -> B.
Proof.
  intros A B HA HNA.
  exfalso.        (* Goal changes from B to False *)
  apply HNA.      (* ~A means A -> False *)
  exact HA.       (* We have A *)
Qed.

(* Alternative: contradiction tactic does this automatically *)
Theorem using_contradiction : forall A B : Prop, A -> ~A -> B.
Proof.
  intros A B HA HNA.
  contradiction.  (* Finds HA and HNA, derives False *)
Qed.

When to use exfalso: When you can derive a contradiction from your hypotheses, but your goal is not False. The tactic exfalso lets you change direction: instead of proving your actual goal, you prove False (which implies everything).

Biconditional: A <-> B

The biconditional A <-> B (read "A if and only if B" or "A iff B") means both A -> B and B -> A. It's actually defined as a conjunction of two implications!

iff.v
(* A <-> B is defined as (A -> B) /\ (B -> A) *)
Print iff.   (* fun A B : Prop => (A -> B) /\ (B -> A) *)

(* To PROVE a biconditional: use split, then prove both directions *)
Theorem iff_example : forall A : Prop, A <-> A.
Proof.
  intro A.
  split.                (* Two subgoals: A -> A and A -> A *)
  - intro H. exact H.   (* First direction *)
  - intro H. exact H.   (* Second direction *)
Qed.

(* To USE a biconditional: destruct it or use rewrite *)
Theorem use_iff : forall A B : Prop, (A <-> B) -> A -> B.
Proof.
  intros A B [HAB HBA] HA.  (* Destruct iff into both implications *)
  apply HAB.                 (* Use the A -> B direction *)
  exact HA.
Qed.

(* Or use rewrite with <-> *)
Theorem rewrite_iff : forall A B : Prop, (A <-> B) -> (A -> B).
Proof.
  intros A B Hiff HA.
  rewrite <- Hiff.    (* rewrite works with <-> too! *)
  exact HA.
Qed.

Biconditional Quick Reference

  • To prove: split, then prove both A -> B and B -> A
  • To use: destruct H as [HAB HBA] to get both implications
  • With rewrite: rewrite H or rewrite <- H works with iff!

Classical vs Intuitionistic Logic

By default, Rocq uses intuitionistic logic. For classical reasoning, import it:

classical.v
(* Import classical logic *)
Require Import Classical.

(* Now we have excluded middle *)
Check classic.   (* forall P : Prop, P \/ ~P *)

(* Double negation elimination *)
Theorem double_neg_elim : forall A : Prop, ~~A -> A.
Proof.
  intros A HNNA.
  destruct (classic A) as [HA | HNA].
  - exact HA.
  - exfalso. apply HNNA. exact HNA.
Qed.

The Drinker's Paradox

"In any non-empty bar, there's someone such that if they're drinking, everyone is drinking."

This sounds paradoxical, but it's true! The trick is in the "if ... then" part. Let's break it down:

The Intuition

Case 1: Everyone is drinking. Pick anyone — the statement "if they're drinking, everyone is drinking" is trivially true.
Case 2: Someone is NOT drinking. Pick that person. The statement "if they're drinking, everyone is drinking" is vacuously true because the premise (they're drinking) is false!

drinker.v
Require Import Classical.

Theorem drinkers_paradox :
  forall (Person : Type) (drinks : Person -> Prop),
  (exists someone : Person, True) ->   (* Bar is non-empty *)
  exists x : Person, drinks x -> forall y : Person, drinks y.
Proof.
  intros Person drinks [someone _].

  (* Key insight: use excluded middle on "everyone drinks" *)
  destruct (classic (forall y, drinks y)) as [Hall | Hnot_all].

  - (* Case 1: Everyone drinks *)
    (* Pick anyone - the implication holds because conclusion is true *)
    exists someone.
    intro _.          (* We don't need the hypothesis *)
    exact Hall.       (* Everyone drinks *)

  - (* Case 2: Not everyone drinks, so someone doesn't drink *)
    (* We need to find that sober person *)
    (* This uses classical logic: ~(forall y, P y) -> exists y, ~P y *)
    assert (exists y, ~drinks y) as [sober Hsober].
    { (* Proof by contradiction using NNPP (double negation) *)
      apply NNPP. intro H.
      apply Hnot_all. intro y.
      apply NNPP. intro Hny.
      apply H. exists y. exact Hny. }

    (* Pick the sober person *)
    exists sober.
    intro Hdrinks.    (* Assume the sober person drinks... *)
    exfalso.          (* ...that's a contradiction! *)
    apply Hsober.     (* They don't drink... *)
    exact Hdrinks.    (* ...but we assumed they do *)
Qed.

Note: This proof requires classical logic (Require Import Classical) because we use the law of excluded middle: either everyone drinks, or not everyone drinks. This is not provable in intuitionistic logic!

Summary

  • /\ — use split to prove, destruct to use
  • \/ — use left/right to prove, destruct cases
  • ~A — defined as A -> False
  • Intuitionistic vs Classical logic matters!
  • exfalso, contradiction — handle contradictions
