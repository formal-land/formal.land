Key Functions Reference

Before diving into the challenges, here are some important functions from the standard library that you'll need:

reference.v
Require Import List.
Import ListNotations.

(* FILTER: keep elements satisfying a predicate *)
(* filter (fun x => x > 2) [1;2;3;4] = [3;4] *)
Fixpoint filter {A : Type} (f : A -> bool) (l : list A) : list A :=
  match l with
  | [] => []
  | h :: t => if f h then h :: filter f t
                     else filter f t
  end.

(* FOLD_RIGHT: combine elements from right to left *)
(* fold_right plus 0 [1;2;3] = 1 + (2 + (3 + 0)) = 6 *)
Fixpoint fold_right {A B : Type} (f : A -> B -> B) (b : B) (l : list A) : B :=
  match l with
  | [] => b
  | h :: t => f h (fold_right f b t)
  end.

(* IN: membership predicate *)
(* In 2 [1;2;3] means 2 is in the list *)
Fixpoint In {A : Type} (x : A) (l : list A) : Prop :=
  match l with
  | [] => False
  | h :: t => h = x \/ In x t
  end.

Proof Strategy Tips

  • Induction on lists: Use induction l as [| h t IHt]
  • If expressions: Use destruct (f h) to case split
  • Inequalities: Look for le_n_S, le_S in Arith
  • Helper lemmas: Prove simpler facts first, then use them

The lia Tactic

lia (Linear Integer Arithmetic) is a powerful decision procedure for goals involving linear arithmetic on integers and natural numbers:

lia_examples.v
Require Import Lia.  (* Import the lia tactic *)

(* lia can solve linear arithmetic goals automatically *)
Goal forall n : nat, n + 0 = n.
Proof. lia. Qed.

Goal forall n m : nat, n <= m -> n <= m + 1.
Proof. lia. Qed.

Goal forall n : nat, 1 <= 2.
Proof. lia. Qed.

(* lia handles inequalities, equalities, and combinations *)
Goal forall a b c : nat, a <= b -> b <= c -> a <= c.
Proof. lia. Qed.

Use lia when your goal involves <=, <, =, +, -, or multiplication by constants. It often solves goals that would take several manual steps!

fold_right vs fold_left

These folds process lists in opposite directions:

  • fold_right f b [1;2;3] = f 1 (f 2 (f 3 b)) — processes from right
  • fold_left f b [1;2;3] = f (f (f b 1) 2) 3 — processes from left

For associative operations like +, both give the same result. For non-associative operations, the results differ!

Import ListNotations.

  length (filter f l) <= length l.
Proof.
  (* Your proof here *)

Qed.

Strategy:

  • Do induction on l
  • In the h :: t case, destruct (f h) to handle both branches of the if
  • When f h = true: filter keeps h, use le_n_S with IH
  • When f h = false: filter drops h, use le_S with IH

Useful lemmas from Arith:

  • le_n_S : forall n m, n <= m -> S n <= S m
  • le_S : forall n m, n <= m -> n <= S m

Why destruct (f h)?

The filter function uses if f h then ... else .... When you destruct (f h), Rocq considers both cases (true and false) and rewrites the if expression accordingly in each branch. This is the standard technique for reasoning about conditionals!

Challenge 4: Pigeonhole Principle (Simplified)

If a list has more elements than there are values in a finite set, there must be duplicates. Here's a simplified version.

challenge4.v
Require Import List Arith.
Import ListNotations.

(* Check if x is in list l *)
Fixpoint In {A : Type} (x : A) (l : list A) : Prop :=
  match l with
  | [] => False
  | h :: t => h = x \/ In x t
  end.

(* Prove: if x is in (l1 ++ l2), then x is in l1 or l2 *)
Theorem in_app_iff : forall (A : Type) (x : A) (l1 l2 : list A),
  In x (l1 ++ l2) <-> In x l1 \/ In x l2.
Proof.
  (* Your proof here *)

Qed.

Remember: <-> means you need to prove both directions!

  • Use split to separate the two directions
  • Do induction on l1 for both directions
  • In the base case, [] ++ l2 = l2 and In x [] = False
  • In the inductive case, unfold In and use destruct on disjunctions

Key insight: In x (h :: t) unfolds to h = x \/ In x t

Bonus Challenge: Dependent Types Preview

Here's a taste of dependent types — vectors (lists with length in the type!).

vectors.v
(* Vectors: lists where the length is part of the type! *)
Inductive vec (A : Type) : nat -> Type :=
  | vnil : vec A 0
  | vcons : forall n, A -> vec A n -> vec A (S n).

Arguments vnil {A}.
Arguments vcons {A n}.

(* Example: a vector of exactly 3 elements *)
Definition three_nums : vec nat 3 :=
  vcons 1 (vcons 2 (vcons 3 vnil)).

(* Head function that CANNOT fail - no empty vector possible! *)
Definition vhead {A : Type} {n : nat} (v : vec A (S n)) : A :=
  match v with
  | vcons _ h _ => h
  end.

Compute vhead three_nums.  (* = 1 *)

(* Try to define vhead for vec A 0 - you can't!
   The type system prevents accessing empty vectors. *)

The Arguments Command

The Arguments command controls which arguments are implicit:

arguments_explained.v
(* After defining vnil and vcons, we make some args implicit *)
Arguments vnil {A}.       (* A is implicit for vnil *)
Arguments vcons {A n}.    (* A and n are implicit for vcons *)

(* Now instead of writing: *)
(* vcons nat 2 1 (vcons nat 1 2 (vcons nat 0 3 (vnil nat))) *)

(* We can write: *)
vcons 1 (vcons 2 (vcons 3 vnil))

(* Rocq infers A=nat and n=2,1,0 automatically! *)

Arguments is useful when you've already defined something and want to retroactively make certain parameters implicit. The curly braces mark which arguments should be inferred by Rocq.

Advanced Tactics Reference

Beyond the core tactics, Rocq has many specialized tactics for common proof patterns. Here's a reference for tactics you'll encounter as you progress.

Goal Manipulation Tactics

goal_tactics.v
(* UNFOLD: expand a definition in the goal *)
Definition double (n : nat) := n + n.

Goal double 3 = 6.
Proof.
  unfold double.    (* Goal becomes: 3 + 3 = 6 *)
  reflexivity.
Qed.

(* FOLD: the opposite of unfold - collapse back *)
Goal 3 + 3 = 6.
Proof.
  fold (double 3).  (* Goal becomes: double 3 = 6 *)
  reflexivity.
Qed.

(* CHANGE: replace goal with a convertible term *)
Goal 2 + 2 = 4.
Proof.
  change (2 + 2 = 2 * 2).  (* Replace 4 with 2*2 *)
  reflexivity.
Qed.

(* REPLACE: substitute equals for equals *)
Goal forall n : nat, n + 0 = n -> (n + 0) + 1 = n + 1.
Proof.
  intros n H.
  replace (n + 0) with n.  (* Creates subgoal: n = n + 0 *)
  - reflexivity.
  - symmetry. exact H.
Qed.

Hypothesis Manipulation

hypothesis_tactics.v
(* CLEAR: remove a hypothesis *)
Goal forall A B : Prop, A -> B -> B.
Proof.
  intros A B HA HB.
  clear HA.         (* Remove HA - we don't need it *)
  exact HB.
Qed.

(* RENAME: rename a hypothesis *)
Goal forall A : Prop, A -> A.
Proof.
  intros A H.
  rename H into evidence.  (* H is now called evidence *)
  exact evidence.
Qed.

(* SPECIALIZE: instantiate a universal hypothesis *)
Goal forall (P : nat -> Prop), (forall n, P n) -> P 42.
Proof.
  intros P Hall.
  specialize (Hall 42).   (* Hall is now P 42 *)
  exact Hall.
Qed.

(* POSE PROOF: add a derived fact to hypotheses *)
Goal forall n : nat, n = n.
Proof.
  intro n.
  pose proof (Nat.add_0_r n) as H.  (* Add n + 0 = n as H *)
  reflexivity.
Qed.

(* GENERALIZE: move a term from goal to hypothesis position *)
Goal 1 + 1 = 2.
Proof.
  generalize 1.     (* Goal becomes: forall n, n + 1 = 2 -- wait, that's wrong! *)
  (* This is useful for strengthening induction hypotheses *)
Abort.

Control Flow Tactics

control_tactics.v
(* TRY: attempt a tactic, do nothing if it fails *)
Goal True.
Proof.
  try reflexivity.  (* Fails silently *)
  exact I.
Qed.

(* REPEAT: apply a tactic until it fails *)
Goal True /\ True /\ True /\ True.
Proof.
  repeat split.     (* Applies split until no more conjunctions *)
  all: exact I.     (* Solve all remaining goals *)
Qed.

(* FIRST: try tactics in order until one succeeds *)
Goal True.
Proof.
  first [exact I | reflexivity | auto].
Qed.

(* ALL: apply tactic to all goals *)
Goal True /\ True.
Proof.
  split.
  all: exact I.     (* Applies to both subgoals *)
Qed.

(* ONLY n: apply tactic to specific goal *)
Goal True /\ False -> True.
Proof.
  intro H. destruct H as [HT HF].
  exact HT.
Qed.

Tactic Combinators Summary

  • t1; t2 — run t2 on all goals created by t1
  • t1 || t2 — try t1, if it fails try t2
  • try t — run t, ignore failure
  • repeat t — run t until it fails
  • first [t1 | t2 | t3] — try in order
  • all: t — run t on all goals

Working with Equality

Equality proofs often require specialized tactics beyond rewrite and reflexivity. Here are the key tactics for equality reasoning.

symmetry and transitivity

eq_basic.v
(* SYMMETRY: flip an equality *)
Goal forall n m : nat, n = m -> m = n.
Proof.
  intros n m H.
  symmetry.         (* Goal: n = m *)
  exact H.
Qed.

(* TRANSITIVITY: chain equalities *)
Goal forall a b c : nat, a = b -> b = c -> a = c.
Proof.
  intros a b c Hab Hbc.
  transitivity b.   (* Two goals: a = b and b = c *)
  - exact Hab.
  - exact Hbc.
Qed.

(* SUBST: substitute everywhere *)
Goal forall n m : nat, n = m -> n + n = m + m.
Proof.
  intros n m H.
  subst.            (* Replaces n with m everywhere *)
  reflexivity.
Qed.

injection: Extracting from Constructors

When two constructor applications are equal, their arguments must be equal. injection extracts these equalities:

injection.v
(* If S n = S m, then n = m *)
Goal forall n m : nat, S n = S m -> n = m.
Proof.
  intros n m H.
  injection H.      (* Introduces n = m as a hypothesis *)
  intro Heq.
  exact Heq.
Qed.

(* With as clause to name the result *)
Goal forall n m : nat, S n = S m -> n = m.
Proof.
  intros n m H.
  injection H as Heq.   (* Directly names it Heq *)
  exact Heq.
Qed.

(* Works with any constructor *)
Require Import List.
Import ListNotations.

Goal forall (A : Type) (x y : A) (xs ys : list A),
  x :: xs = y :: ys -> x = y /\ xs = ys.
Proof.
  intros A x y xs ys H.
  injection H as Hxy Hxsys.   (* Get both equalities *)
  split.
  - exact Hxy.
  - exact Hxsys.
Qed.

discriminate: Different Constructors Can't Be Equal

If you have an equality between different constructors, that's a contradiction. discriminate solves such goals:

discriminate.v
(* 0 can never equal S n *)
Goal forall n : nat, 0 = S n -> False.
Proof.
  intros n H.
  discriminate H.   (* Contradiction! Different constructors. *)
Qed.

(* This works for any impossible equality *)
Goal forall P : Prop, true = false -> P.
Proof.
  intros P H.
  discriminate H.   (* Proves any goal from contradiction *)
Qed.

(* Useful in case analysis *)
Goal forall n : nat, S n <> 0.
Proof.
  intros n H.
  discriminate H.
Qed.

inversion: Analyzing How a Value Was Built

inversion is like a "super destruct" that also derives equalities from how a value must have been constructed:

inversion.v
(* Recall the sorted predicate from earlier *)
Inductive sorted : list nat -> Prop :=
  | sorted_nil : sorted []
  | sorted_one : forall x, sorted [x]
  | sorted_cons : forall x y l,
      x <= y -> sorted (y :: l) -> sorted (x :: y :: l).

(* inversion deduces facts from how sorted could have been proved *)
Goal forall x y l, sorted (x :: y :: l) -> x <= y.
Proof.
  intros x y l H.
  inversion H.      (* H must have been built with sorted_cons *)
  (* Gives us: H1 : x <= y and H2 : sorted (y :: l) *)
  exact H1.
Qed.

(* Another example: le (less-than-or-equal) *)
Goal forall n : nat, n <= 0 -> n = 0.
Proof.
  intros n H.
  inversion H.      (* Only way n <= 0 is if n = 0 *)
  reflexivity.
Qed.

(* inversion can solve impossible cases *)
Goal sorted [3; 2; 1] -> False.
Proof.
  intro H.
  inversion H.      (* sorted_cons requires 3 <= 2 *)
  inversion H3.     (* and sorted [2; 1] requires 2 <= 1 *)
  lia.              (* But 2 <= 1 is false! *)
Qed.

When to Use Each Tactic

  • injection — when C x = C y, extract x = y
  • discriminate — when C1 x = C2 y (different constructors)
  • inversion — when you have H : P x for an inductive predicate and need to know how H was constructed
  • subst — when you have x = e and want to replace all x with e

f_equal: Applying Functions to Both Sides

f_equal.v
(* If a = b, then f a = f b *)
Goal forall n m : nat, n = m -> S n = S m.
Proof.
  intros n m H.
  f_equal.          (* Goal becomes n = m *)
  exact H.
Qed.

(* Works with any function *)
Goal forall (A B : Type) (f : A -> B) (x y : A),
  x = y -> f x = f y.
Proof.
  intros A B f x y H.
  f_equal.
  exact H.
Qed.

(* f_equal can handle multiple arguments *)
Goal forall a b c d : nat, a = c -> b = d -> a + b = c + d.
Proof.
  intros a b c d Hac Hbd.
  f_equal.          (* Two subgoals: a = c and b = d *)
  - exact Hac.
  - exact Hbd.
Qed.

Proof Automation Deep Dive

Rocq provides several tactics for automating proofs. Understanding when to use each one will make you much more productive.

auto and Hint Databases

auto_hints.v
(* auto searches for proofs using a hint database *)
Goal forall A B C : Prop, A -> (A -> B) -> (B -> C) -> C.
Proof.
  auto.   (* Chains intro and apply automatically *)
Qed.

(* auto has a depth limit (default 5) *)
(* For deeper proofs, increase it: *)
Goal forall A B C D E F : Prop,
  A -> (A->B) -> (B->C) -> (C->D) -> (D->E) -> (E->F) -> F.
Proof.
  auto 10.   (* Increase search depth to 10 *)
Qed.

(* You can add your own lemmas to auto's database *)
Lemma my_lemma : forall n : nat, n + 0 = n.
Proof. intro n. lia. Qed.

(* Register it as a hint *)
#[local] Hint Resolve my_lemma : core.

Goal 5 + 0 = 5.
Proof.
  auto.   (* Now uses my_lemma! *)
Qed.

eauto: Extended Auto

eauto.v
(* eauto can handle existential variables *)
Goal exists n : nat, n = 0.
Proof.
  eauto.   (* Finds the witness automatically! *)
Qed.

(* eauto is more powerful but slower than auto *)
Goal forall (A : Type) (P : A -> Prop) (x : A),
  P x -> exists y, P y.
Proof.
  eauto.   (* Introduces existential, finds witness *)
Qed.

intuition and tauto

intuition_tauto.v
(* tauto: decision procedure for propositional logic *)
Goal forall A B : Prop, A /\ B -> B /\ A.
Proof. tauto. Qed.

Goal forall A B : Prop, A \/ B -> B \/ A.
Proof. tauto. Qed.

Goal forall A B C : Prop, (A -> B -> C) -> (A /\ B -> C).
Proof. tauto. Qed.

(* intuition: like tauto but also simplifies *)
Goal forall A B : Prop, A /\ (B \/ A) -> A.
Proof. intuition. Qed.

(* intuition can call another tactic on leftover goals *)
Goal forall n : nat, n = n /\ n + 0 = n.
Proof.
  intuition lia.   (* tauto handles /\, lia handles arithmetic *)
Qed.

Debugging Proofs

When proofs go wrong, these techniques help you understand what's happening.

Inspecting the Proof State

debugging.v
(* Show: display current proof state (IDE usually does this) *)
Goal forall n : nat, n = n.
Proof.
  intro n.
  Show.             (* Explicitly show proof state *)
  reflexivity.
Qed.

(* Show Proof: see the proof term being constructed *)
Goal forall A : Prop, A -> A.
Proof.
  intros A H.
  Show Proof.       (* Shows: (fun (A : Prop) (H : A) => ?Goal) *)
  exact H.
  Show Proof.       (* Shows: (fun (A : Prop) (H : A) => H) *)
Qed.

(* Info: see what a tactic did *)
Goal forall A B : Prop, A -> B -> A /\ B.
Proof.
  intros A B HA HB.
  Info 1 auto.      (* Shows the tactics auto used *)
Qed.

Pro Debugging Tips

  • Read error messages carefully — they often tell you exactly what's wrong
  • Use Search liberally — there's probably a lemma for what you need
  • Try simpl or cbn — often the goal just needs simplification
  • Check parentheses — A -> B -> C is A -> (B -> C)
  • Strengthen your induction hypothesis — sometimes you need to prove more to prove less!

Sections and Modules

For organizing larger developments, Rocq provides sections (for local assumptions) and modules (for namespacing and abstraction).

Sections: Local Contexts

sections.v
(* Sections let you introduce local assumptions *)
Section MySection.

  (* Variables are local to this section *)
  Variable A : Type.
  Variable P : A -> Prop.

  (* Hypotheses are assumed within the section *)
  Hypothesis H : forall x, P x.

  (* Theorems can use A, P, H without quantifying *)
  Theorem example1 : forall x : A, P x.
  Proof. exact H. Qed.

  Theorem example2 : forall x y : A, P x /\ P y.
  Proof.
    intros x y. split.
    - apply H.
    - apply H.
  Qed.

End MySection.

(* After the section, theorems are generalized! *)
Check example1.
(* example1 : forall (A : Type) (P : A -> Prop),
              (forall x, P x) -> forall x, P x *)

Check example2.
(* example2 : forall (A : Type) (P : A -> Prop),
              (forall x, P x) -> forall x y, P x /\ P y *)

Modules: Namespaces

modules.v
(* Modules group related definitions *)
Module NatUtils.

  Definition double (n : nat) := n + n.
  Definition triple (n : nat) := n + n + n.

  Lemma double_plus : forall n, double n = n + n.
  Proof. reflexivity. Qed.

End NatUtils.

(* Access with qualification *)
Compute NatUtils.double 5.   (* = 10 *)

(* Or import the module *)
Import NatUtils.
Compute double 5.            (* = 10 *)

(* Module Types define interfaces *)
Module Type MONOID.
  Parameter T : Type.
  Parameter unit : T.
  Parameter op : T -> T -> T.
  Axiom unit_left : forall x, op unit x = x.
  Axiom unit_right : forall x, op x unit = x.
  Axiom assoc : forall x y z, op (op x y) z = op x (op y z).
End MONOID.

(* Modules can implement interfaces *)
Module NatAddMonoid <: MONOID.
  Definition T := nat.
  Definition unit := 0.
  Definition op := Nat.add.
  (* ... prove the axioms ... *)
End NatAddMonoid.

When to Use Sections vs Modules

  • Sections: When you want to avoid repeating type parameters and hypotheses across many lemmas
  • Modules: When you want namespacing, separate compilation, or to implement abstract interfaces

Sections are "transparent" — after the section ends, everything is visible. Modules can hide implementation details.

Tactic Learning Strategy

Don't try to memorize all tactics at once! Start with the core set: intro, apply, exact, destruct, induction, simpl, rewrite, reflexivity.

As you encounter new proof patterns, gradually add more tactics to your toolkit. Use Search to find useful lemmas, and don't hesitate to check the Rocq reference manual for detailed descriptions of each tactic!
